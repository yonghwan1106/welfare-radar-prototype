# 복지 레이더 - User Journey

## 사용자 여정 문서

## 1. 개요

본 문서는 복지 레이더 시스템의 주요 사용자 시나리오를 정의합니다.
각 여정은 사용자의 목표, 단계별 행동, 시스템 반응, 감정 곡선을 포함합니다.

### 주요 사용자

| 페르소나 | 역할 | 주요 목표 |
|----------|------|----------|
| **김주무관** | 복지 담당 공무원 | 위기가구 발굴 및 복지 연결 |
| **박과장** | 관리자 | 팀 성과 모니터링 및 보고 |

## 2. Journey Map 프레임워크

### 2.1 Touchpoint 정의

사용자가 시스템과 상호작용하는 모든 접점입니다.

| Touchpoint | 설명 | 주요 화면 |
|------------|------|----------|
| **로그인** | 시스템 진입점 | 로그인 페이지 |
| **대시보드** | 현황 파악의 허브 | 대시보드 |
| **목록 조회** | 데이터 탐색 | 위기가구 목록, 복지서비스 목록 |
| **상세 조회** | 심층 정보 확인 | 가구 상세, 서비스 상세 |
| **액션 수행** | 업무 처리 | 안내 발송, 메모 추가, 상태 변경 |
| **알림 수신** | 푸시/배지 알림 | 전체 화면 (헤더) |
| **리포트 조회** | 성과 분석 | 통계 페이지 |

| Channel | 사용 상황 | 최적화 우선순위 |
|---------|----------|:---------------:|
| **Desktop Web** | 사무실 근무 시 | P0 |
| **Tablet Web** | 현장 방문 시 | P1 |
| **Mobile Web** | 이동 중, 긴급 확인 | P1 |
| **Push 알림** | 긴급 상황 통보 | P2 |

|:----:|:------:|----------|------|

## Journey 1: 긴급 위기가구 발견 및 대응

### 시나리오
> 김주무관이 출근하여 시스템을 확인하고, 긴급 위기가구를 발견하여 복지 서비스를 연결하는 과정

### 사용자 목표
- 긴급 위기가구를 신속히 파악
- AI 분석을 통해 위기 원인 이해
- 적합한 복지 서비스 연결
- 선제 안내 발송

### 단계별 상세

#### Step 1: 로그인 및 대시보드 진입

**사용자 행동**
```
- 복지 레이더 시스템 접속
- 로그인 (ID/PW 또는 공무원 인증)
- 대시보드 화면 확인
```

**시스템 반응**
```
- 오늘의 긴급 현황 알림 배지 표시
- "긴급 위기가구 3건" 알림 팝업
- KPI 카드에 주요 지표 표시
```

**화면 요소**
- 🔴 긴급 알림 배지 (3)
- KPI: 사각지대 감소율, 연결률, 평균 발견시간
- 오늘의 할 일 목록 (긴급 우선)

**Pain Points**
- 매일 로그인 과정이 번거로움
- 알림이 많으면 무엇부터 해야 할지 혼란

**Opportunities**
- 자동 로그인 또는 SSO 지원
- 긴급도에 따른 명확한 우선순위 표시

**사용자 감정**: 😮 "오늘 긴급 건이 있네, 바로 확인해야겠다"

---

#### Step 2: 긴급 위기가구 목록 확인

**사용자 행동**
```
- "긴급 위기가구 3건" 클릭
- 목록에서 위기스코어 높은 순으로 정렬 확인
- 가장 위험한 "박OO" 가구 선택
```

**시스템 반응**
```
- 긴급 등급 필터가 적용된 목록 표시
- 위기스코어 87점 "박OO" 가구가 최상단
- 간략 정보: 67세, 1인가구, 경제+고립 위기
```

**화면 요소**
| 이름 | 연령 | 유형 | 위기등급 | 스코어 | 주요위기 |
|------|------|------|:--------:|:------:|----------|
| 박OO | 67세 | 1인 | 🔴 긴급 | 87 | 경제, 고립 |
| 이OO | 72세 | 1인 | 🔴 긴급 | 82 | 건강, 고립 |
| 최OO | 55세 | 1인 | 🔴 긴급 | 80 | 경제 |

**Pain Points**
- 비슷한 스코어일 때 누구부터 처리해야 할지 고민
- 목록이 길면 스크롤 필요

**Opportunities**
- AI 권장 처리 순서 제시
- 페이지네이션 최적화

**사용자 감정**: 🤔 "87점이면 심각하네. 자세히 봐야겠다"

---

#### Step 3: 가구 상세 페이지 - AI 분석 확인

**사용자 행동**
```
- 박OO 가구 상세 페이지 진입
- 6차원 레이더 차트로 위기 분포 확인
- AI 분석 결과 읽기
- 위기 요인별 영향도 파악
```

**시스템 반응**
```
[6차원 레이더 차트]
- 경제적 위기: 95점 (매우 높음)
- 건강 위기: 70점 (높음)
- 사회적 고립: 85점 (매우 높음)
- 주거 위기: 45점 (보통)
- 가족 위기: 30점 (낮음)
- 행정 위기: 60점 (높음)

[AI 분석 요약]
"전기요금 4개월 연속 체납과 병원 방문 3개월 중단이
주요 위기 요인으로 분석됩니다. 긴급 개입이 필요합니다."

[핵심 위기 요인]
1. 전기요금 체납 4개월 (누적 45만원) - 영향도 35%
2. 병원 방문 중단 3개월 - 영향도 28%
3. 1인 독거 고령자 - 영향도 22%
4. 국민연금 미납 2개월 - 영향도 15%
```

**Pain Points**
- AI 분석 결과가 맞는지 확인할 방법 없음
- 데이터 출처에 대한 의문

**Opportunities**
- 각 위기 요인의 데이터 출처 명시
- "이 분석이 도움이 됐나요?" 피드백 수집

**화면 요소**
- 레이더 차트 (6축)
- AI 분석 카드 (요약, 요인, 권장조치)
- 위기 지표 타임라인 (체납 추이 그래프)

**사용자 감정**: 😊 "왜 위험한지 명확하게 이해됐다. 뭘 해야 할지 알겠어"

- 자격 확인에 필요한 정보 명시
- 복수 서비스 동시 안내 기능

**사용자 감정**: 😊 "자격 요건도 맞고, 적합도도 높네. 바로 안내하자"

```

│     [대시보드로 이동]                       │
│                                             │
└─────────────────────────────────────────────┘
```

**온보딩 완료 후 지원**
```
💡 도움이 필요하신가요?

• [도움말] - 기능별 가이드
• [FAQ] - 자주 묻는 질문
• [튜토리얼 다시 보기]
• [헬프데스크] - 1:1 문의

첫 주간 동안 매일 오전에
일일 팁을 알려드릴게요!
```

**Pain Points**
- 튜토리얼 끝나면 혼자 해야 해서 막막
- 도움 받을 곳 모름

**Opportunities**
- 첫 주간 특별 서포트
- 맥락에 맞는 도움말 자동 표시
- 선배 담당자 멘토링 연결

| 상세 분석 | AI 분석 결과 확인 | 가구 상세 1회 조회 |
| 안내 발송 | 복지 안내 발송 | 발송 1회 완료 |

## 핵심 UX 원칙

### 1. 긴급 상황 즉시 인지
- 로그인 시 긴급 알림 즉시 표시
- 빨간색 배지와 팝업으로 주의 환기
- 긴급도에 따른 명확한 시각적 구분

### 2. AI 분석의 투명성
- 왜 위험한지 명확히 설명
- 영향도 퍼센트로 신뢰도 제공
- 데이터 출처 명시

### 3. 최소 클릭 원칙
- 목록 → 상세 → 안내 발송: 3단계 이내
- 빠른 액션 버튼으로 단축
- 자주 쓰는 기능 우선 배치

### 4. 컨텍스트 유지
- 이전 활동 기록 자동 표시
- 다음 권장 액션 제안
- 관련 정보 연결

### 5. 피드백 명확화
- 모든 액션에 성공/실패 피드백
- 발송 결과 추적 가능
- 에러 시 명확한 원인과 대안 제시

### 6. 점진적 학습 지원
- 처음 사용자를 위한 온보딩
- 맥락에 맞는 도움말
- 연습 모드로 안전한 학습

### 7. 접근성 우선
- 키보드 네비게이션 완벽 지원
- 스크린리더 호환
- 고대비 모드 제공
