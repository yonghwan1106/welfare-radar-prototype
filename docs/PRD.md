# 복지 레이더 - PRD (Product Requirements Document)

## AI 예측 기반 복지 사각지대 선제 발굴 시스템

---

## Executive Summary

### 한 줄 요약
**복지 레이더**는 AI가 47종 위기 정보를 분석하여 복지 사각지대 가구를 선제적으로 발굴하고, 450개 복지서비스 중 최적을 매칭하여 담당자의 선제 안내를 지원하는 시스템입니다.

### 핵심 문제
| 문제 | 현황 | 영향 |
|------|------|------|
| 복지 사각지대 | 연간 165만 가구 | 고독사 연 3,300명 |
| 늦은 발견 | 평균 14일 소요 | 위기 악화, 비용 증가 |
| 정보 과부하 | 450개 서비스, 2,300개 기준 | 담당자 소진, 오연결 |

### 솔루션
```
┌─────────────────────────────────────────────────────────────────┐
│                     복지 레이더 솔루션                           │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   [47종 데이터]  →  [AI 위기 분석]  →  [담당자 대응]  →  [시민]   │
│                                                                 │
│   건강보험 체납        6차원 스코어링     선제 안내 발송    복지    │
│   전기 체납           위기 예측           서비스 매칭     수혜    │
│   병원 방문 중단       복합 위기 탐지      방문 상담       지원    │
│   ...                                                          │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 기대 효과
| 지표 | 현재 → 목표 | 개선율 | 사회적 가치 |
|------|-------------|:------:|-------------|
| 복지 사각지대 | 165만 → 82만 가구 | **-50%** | 82만 가구 추가 보호 |
| 위기 발견 시간 | 14일 → 3일 | **-79%** | 조기 개입으로 위기 악화 방지 |
| 복지 연결률 | 60% → 85% | **+42%** | 적합한 서비스 연결 |
| 고독사 예방 | - | - | **연간 1,500명+** |
| ROI | - | - | **50배** (50억 → 2.5조) |

### 타겟 사용자
- **Primary**: 복지 담당 공무원 (전국 약 3만 명)
- **Secondary**: 복지정책과 관리자
- **Tertiary**: 복지 수혜 시민

## 3. 제품 비전

> **"몰라서 못 받는 복지, AI가 먼저 찾아갑니다"**

복지 레이더는 신청을 기다리지 않고, AI가 위기 가구를 선제적으로 발굴하여 필요한 복지 서비스를 먼저 연결하는 시스템입니다.

### 3.1 핵심 가치

| 가치 | 설명 | 기대 효과 |
|------|------|----------|
| **선제 발굴 (Discover)** | AI가 위기 가구를 먼저 찾아감 | 사각지대 50% 감소 |
| **정확한 매칭 (Connect)** | 450개 복지서비스 중 최적 연결 | 연결률 42% 향상 |
| **지속 보호 (Protect)** | 재위기 모니터링으로 지속 관리 | 재위기율 60% 감소 |

### 3.2 목표 지표 (KPI)

| 지표 | 현재 | 목표 | 개선율 |
|------|------|------|--------|
| 복지 사각지대 | 100% | 50% | **-50%** |
| 위기 발견 시간 | 14일 | 3일 | **-79%** |
| 복지 연결률 | 60% | 85% | **+42%** |
| 오분류율 | 14% | 5% | **-64%** |
| 고독사 예방 | - | 연 1,500명+ | - |

## 9. 에러 처리 & 엣지 케이스

### 9.1 에러 처리 정책

#### 데이터 로딩 실패
| 상황 | 사용자 메시지 | 시스템 동작 |
|------|--------------|------------|
| API 타임아웃 | "데이터를 불러오는 중입니다..." | 3회 재시도 후 오류 표시 |
| 네트워크 오류 | "네트워크 연결을 확인해주세요" | 오프라인 캐시 데이터 표시 |
| 서버 오류 (500) | "일시적인 오류가 발생했습니다" | 관리자 알림, 재시도 버튼 |
| 권한 오류 (403) | "접근 권한이 없습니다" | 로그인 페이지로 이동 |

#### 발송 실패 처리
| 상황 | 대응 | 복구 액션 |
|------|------|----------|
| 카카오톡 발송 실패 | "발송 실패 (수신 거부)" | 대체 채널(문자) 제안 |
| 문자 발송 실패 | "발송 실패 (번호 오류)" | 연락처 확인 요청 |
| 대량 발송 실패 | "N건 발송 실패" | 실패 건 별도 목록 제공 |

#### 입력 유효성 검증
| 필드 | 검증 규칙 | 에러 메시지 |
|------|----------|------------|
| 연락처 | 010-XXXX-XXXX 형식 | "올바른 연락처 형식으로 입력해주세요" |
| 메모 | 최대 500자 | "메모는 500자까지 입력 가능합니다" |
| 검색어 | 최소 2자 | "2자 이상 입력해주세요" |

### 9.2 엣지 케이스 목록

| 케이스 | 설명 | 처리 방안 |
|--------|------|----------|
| 데이터 없음 | 필터 결과가 0건 | "조건에 맞는 가구가 없습니다" + 필터 초기화 버튼 |
| 중복 발송 | 같은 서비스 재발송 시도 | "이미 발송된 안내입니다. 재발송하시겠습니까?" |
| 동시 편집 | 같은 가구 동시 수정 | 마지막 저장 우선 + 충돌 알림 |
| 긴 텍스트 | AI 분석 결과가 매우 긴 경우 | 요약 표시 + "더보기" 확장 |
| 특수문자 | 이름에 특수문자 포함 | 표시는 가능, 검색 시 이스케이프 |
| 탈퇴 담당자 | 담당자가 시스템 탈퇴 | "담당자 미배정" 표시 + 재배정 알림 |
| 과거 데이터 | 1년 이상 된 기록 | 아카이브 처리 + 필요시 조회 가능 |

## 용어집 (Glossary)

| 용어 | 정의 |
|------|------|
| **복지 사각지대** | 복지 서비스가 필요하지만 받지 못하는 가구 또는 개인 |
| **위기 스코어** | 6차원 위기 요소를 종합한 0-100점 척도. 높을수록 위험 |
| **복합 위기** | 2개 이상의 위기 차원에서 동시에 높은 점수를 보이는 상태 |
| **선제 발굴** | 신청을 기다리지 않고 위기 가구를 먼저 찾아내는 것 |
| **선제 안내** | 위기 가구에게 복지 서비스를 먼저 알려주는 것 |
| **연합학습** | 데이터를 중앙에 모으지 않고 분산된 상태에서 AI를 학습하는 기술 |
| **기준 중위소득** | 전체 가구를 소득순으로 나열할 때 정중앙 가구의 소득 |
| **긴급복지지원** | 위기 상황에 처한 저소득층에게 신속히 지원하는 복지 제도 |
| **행복e음** | 사회보장정보시스템. 복지 급여 관리용 공공 시스템 |
| **Privacy-Preserving AI** | 개인정보를 보호하면서 AI를 활용하는 기술의 총칭 |
